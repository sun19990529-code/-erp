第一部分：本地开发机部署

如果您需要将本旗舰版系统纯离线或在其他局域网服务器/开发机上全新部署，请严格遵循以下执行流：

1. 基础节点环境准备 (Node.js)

新电脑或服务器必须具备基础的 JavaScript 运行引擎：

  • 访问 Node 官方下载站并安装 Node.js（推荐下载 v18.x 或 v20.x 的 LTS 长期支持版）。
  • 安装过程中请一直点击"下一步"，切勿更改默认环境变量的注入配置。

✅ 验证步骤：按下 Win + R 键，输入 cmd 打开终端，敲入如下指令：

node -v
# 若输出版本号如 v18.17.0 等字样则环境绿灯放行。

2. 工程文件迁移与装载

  • 将整个 erp-mes-system 工程文件夹通过 U 盘、坚果云、FTP 或网络传输等手段完整拷贝到目标电脑中。
  • ⚡ 性能向提示：拷贝前先删去 backend/node_modules 与 frontend/node_modules，体积可直降 90% 以上。

3. 全局总装与初始化（核心）

  • 找到并双击运行 install.bat 引导器文件。
  • 该装配脚本将自动完成以下工作：
    1. 拦截并检测本机 Node 环境。
    2. 潜入 backend 目录拉取编译 Windows 二进制依赖。
    3. 潜入 frontend 目录拉取依赖并触发 Vite 编译（npm run build）。
  • 静待提示 "安装完成！" 字样即可。

4. 点火运行服务端

  • 方式一（管理员启动，推荐）：双击 server-start.bat，自动请求 UAC 管理员提权，优先使用 PM2 托管服务
  • 方式二（普通启动）：双击运行 start.bat，直接启动 Node 进程
  • 随即会唤醒常驻控制台进程，并自动打开浏览器。
  • 🔗 默认路由为：http://localhost:3198

⚠️ 警告：该控制台承载了所有业务接口和数据库线程，请绝对不要关闭该黑窗口！

5. 优雅熄火与清理

  • 双击项目目录下的 stop.bat，平滑终止所有进程。

第二部分：远程服务器部署

适用于将系统部署到外网服务器（Windows Server / Windows 主机），实现从开发机一键推送更新。

1. 服务器首次准备

1.1 安装 Node.js

同上述第一部分步骤 1。

1.2 初始化服务器环境

将 deploy 文件夹拷贝到服务器项目目录后，以 管理员身份 打开 PowerShell 运行：

# 允许执行脚本（仅需一次）
Set-ExecutionPolicy RemoteSigned -Scope LocalMachine -Force

# 运行初始化脚本（安装 OpenSSH Server + PM2 + 配置防火墙）
.\deploy\server-init.ps1

该脚本自动完成：

  • ✅ 安装 OpenSSH Server（远程部署通道）
  • ✅ 启动 SSH 服务并设为开机自启
  • ✅ 防火墙放行 22 端口
  • ✅ 安装 PM2 进程管理器

1.3 首次部署与启动

# 安装后端依赖
cd F:\erp-mes-system\backend
npm install --production

# 初始化数据库
node -e "require('./database')"

# 构建前端（首次需要）
cd ..\frontend
npm install
npm run build

# 用 PM2 启动服务
cd ..\backend
pm2 start server.js --name erp
pm2 save

1.4 验证

  • 浏览器访问 http://localhost:3198
  • 默认账号：admin / 密码：admin123

2. 开发机一键部署（日常更新）

在开发机改完代码后，只需一行命令即可自动同步到服务器：

.\deploy\deploy.ps1

脚本自动完成 7 个步骤：

步骤 说明
[1/7] Build frontend 本地构建前端
[2/7] Package 打包（自动排除数据库/node_modules/.git）
[3/7] Upload SCP 加密传输到服务器
[4/7] Extract 服务器端解压覆盖代码
[5/7] Install deps 远程安装后端依赖
[6/7] Restart PM2 自动重启服务
[7/7] Check updates 检查依赖更新

安全保障：数据库文件（*.db / *.db-shm / *.db-wal）永远不会被覆盖，服务器生产数据完全安全。

3. PM2 常用命令

在服务器 PowerShell 中使用：

命令 用途
pm2 status 查看服务运行状态
pm2 restart erp 重启服务
pm2 stop erp 停止服务
pm2 logs erp 查看运行日志
pm2 logs erp --lines 50 查看最近 50 行日志

4. 数据库备份与恢复

备份

copy F:\erp-mes-system\backend\mes.db "F:\备份\mes-$(Get-Date -Format 'yyyyMMdd').db"

恢复

# 1. 必须先停服务！
pm2 stop erp

# 2. 替换数据库
copy "F:\备份\mes-20260326.db" "F:\erp-mes-system\backend\mes.db" -Force

# 3. 删除 WAL 残留（关键！否则数据会被旧缓存覆盖）
del F:\erp-mes-system\backend\mes.db-shm -ErrorAction SilentlyContinue
del F:\erp-mes-system\backend\mes.db-wal -ErrorAction SilentlyContinue

# 4. 重启服务
pm2 start erp

⚠️ 切记：恢复数据库时必须先停服务，否则 SQLite 文件锁定导致恢复无效。

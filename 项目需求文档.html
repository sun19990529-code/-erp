铭晟管理系统 (ERP-MES) - 项目需求文档 (PRD)

文档版本：v1.5.1
更新日期：2026年3月23日

一、 项目背景与业务目标

为了解决生产制造与进销存管理中数据割裂的问题，打破接单、采购、排产、品控与仓储各环节的信息孤岛，本机组建了「铭晟管理系统」。该系统是一套深度融合 **ERP（企业资源计划）**与 **MES（制造执行系统）**核心功能的轻量级、智能化的业务闭环软件。

核心业务目标：

  1. 订单驱动打通：实现"销售订单 ➔ 自动下发生产排产 ➔ 工序流转与委外 ➔ 入库及发货"的全局联动，减少人工数据搬运。
  2. 闭环防重与底线警戒预警：防止双重入库、质检不合格阻断生产、库存低水位自动缺料报警并建议采购。具备物料低线预警熔断逻辑。
  3. 高可视化大盘与追溯：利用现代前端架构及数据图表工具，透视全厂业务健康度与时间流维度的排程走势（APS）；同时拥有从物料直至成品的严格入批出批的底层追溯矩阵。

二、 核心业务生命周期（业务蓝图）

2.1 销售到出货流转（Order-to-Cash）

  • 接到客户【销售订单】 ➔ 【生成生产工单】 ➔ 车间按工序【领料出库】 ➔ [生产流转及可能发生外协委外] ➔ 生产完毕转入【成品检验】 ➔ 合格后【自动/手动成品入库】 ➔ 【销售订单完成】驱动【自动成品出库】。

2.2 采购到入库流转（Procure-to-Pay）

  • 系统侦测缺料库存报警 ➔ 采购员编制【采购订单】 ➔ 供应商发货到厂 ➔ 【采购入库】（联动采购单号） ➔ 物料执行【入库检验】 ➔ 检验合格后，自动增加全局可用库存。

三、 功能模块需求池

1. 系统全景看板 (Dashboard)

  • 核心数据抛跳：常年悬挂当前系统的"待处理订单"、"生产中单据"、"待处理质检"和"报警异常事件"基础统计。
  • 业务时序看板：利用双色渐变折线分布图监控厂区"近 7 天销售接单量与车间排产量"的对比走势。
  • 全局管态健康与安全阀检测：具备动态环块截面图与底水位越界警示。只要有物资被划破底线阈值，就会在数据中心面板激起红色预警模块，允许管理层超快查阅亏空清单。
  • APS 制程推演系统(Gantt)：将车间作业转化为了以时间作为物理横轴的可视化甘特图控件，允许直接框定任务周期；以"进度、延期、挂起"的多色条带宏观调控所有的运作中心并带有一键报工映射。

2. 基础数据中台 (MDM)

这是支持业务开展的根基静态数据。

  • 产品池管理：分为"原材料"、"半成品"、"成品"。其中成品必须支持工序级 BOM 配置与指定流转卡绑定。
  • 往来单位库：沉淀"供应商（支持资质和等级）"、"客户（信用评级管理）"。
  • 组织结构设置：按部门划分，按角色定义原子权限，并支持内部子业务人员与对接外部供应商的人员隔离。

3. 智能仓储与溯源防伪 (WMS)

  • 库存雷达与水位器：支持原材料、半成品与成品按条形码/产品大区即时跟踪状态，设立安全库存下限值并触发熔断警告。
  • 深度批次控制流 (Batch Traceability)：彻底放弃粗颗粒度的简单累加；现在所有仓配操作强制具备批次附魔——每一次采购入库、自动完工入库都将被贴上类似 RK-XXX 或 PRD-XXX 的唯一身世码；而每一次调配、出库也要求精密选取消耗批次。若未人工接管，则系统将按照底层默认约束的 先入先出 (FIFO) 原则无情核销老化库存。
  • 扫码枪数字孪生站：全局配备唯一的 扫码收银台 中枢，只要监听外接的 USB 扫描设备射入单号，直接光速分摊对应的工作流拦截器（例如自动跳转某笔采购单审批界面，或工单流转界面），提升10倍人效。

4. 销售订单中心 (OMS)

  • 支持业务员新建包含不同 SKU 子层级的销售合同订单，具备状态监控功能（待确认 -> 生产中 -> 已完成）。
  • 支持基于当前订单的交期测算与一键"下达排产工单指令"。

5. 车间执行系统 (MES)

  • 工单生成与追踪：继承销售单明细转化为车间制造指令。
  • 工序级报工：支持车间多达近十种专属工序（如轧机、校直、抛光、激光等）分段管控，完成一步流转下一步。
  • 工序物料绑定与自动消耗（v1.4.0）：支持为每道工序配置所需原材料及单位用量；首道工序报工时自动按产出量乘以单位用量计算消耗量，从原材料仓库按 FIFO 批次扣减。库存不足时阻断报工并弹出缺料提示。
  • 领料与物料就绪：监控前置齐套率。若原材料库已出齐，则工单自亮起"材料就绪"绿灯。
  • 一键外协：车间指定工序遇产能瓶颈，勾选"委外"自动派生委外加工单，转入委外质检闭环。
  • 完工数量同步（v1.4.0）：最终工序完成时自动将产出数量回写生产工单的完工数量字段。
  • 状态白名单防护（v1.5.0）：全部 6 个模块的状态更新接口均实施白名单校验，防止非法状态转换。

6. 质量检验保障 (QMS)

所有物料节点的强制拦截器。

  • 多维度检验面：提供"采购买入检验"、"生产过程巡检"、"委外回厂检验"以及"最终成品出厂检验"。
  • 不合格熔断：巡检遇异常，自动挂起当笔工单并将状态冻结为"质检暂停"，直到管理层处理复查合格。

7. 采购与委外跟踪

  • 动态采购建议：根据未完成的未投料销售订单所需物料减去现有静置库存，自动输出包含缺口的采购建议量。
  • 发退料跟踪：监控委外供应商是否已经领料，加工品是否全数回厂质检。

四、 非功能性与技术需求

4.1 技术架构与选型

  • 前端体系：基于 Vite 构建工具重度工程化。使用现代化的单页应用标准 React 19。UI 层以 Tailwind CSS 3/4 原子类构建玻璃流体质感的极简企业级界面；看板监控池集成轻量级图表引擎 Recharts。
  • 后端体系：由 Node.js + Express 运行轻量的 RESTful API 路由容器层；利用内存/文件跨平台双解兼容的 better-sqlite3 提供业务关系解耦和数据落盘保障（WAL 模式）。
  • 通信与交付：前后端解耦部署但融合发布。提供 .bat CLI 自动探针化脚本（检测环境、拉依赖包、动态编译构建 Vite 的 dist 包并用 Node 服务挂载同源输出）。

4.2 性能与安全标准

  1. 加载速率：得益于前端模块剥离和生产级 Vite 构建，网页载入从几秒白屏延迟级提升至毫秒级无感渲染。
  2. 加密存储设计：管理员重置或新增用户凭证时利用强制 bcryptjs 执行带盐强哈希非明文下沉数据库；登录时自动识别旧明文密码并升级。
  3. 查询防抖与检索优化：前端表单应用了防抖钩子拦截无意义的连续击键搜索；SQLite 后端创建了关键业务订单的多列索引，确保存量几十万单时的响应在 500ms 即刻内。
  4. 数据一致性保障（v1.4.0+）：通过事务封装确保库存扣减与报工记录的原子性；res.json 统一在事务外部调用，消除事务回滚后仍返回成功的数据一致性隐患。
  5. API 权限体系（v1.4.1）：实现接口级 RBAC 权限中间件覆盖全部 51+ 路由端点，角色/权限/用户管理写操作限制为 system_admin 专属，防止越权操作。
  6. 访问控制（v1.4.1）：CORS 限制局域网来源、Express Rate Limiting IP 限流（300次/分钟）、JWT 令牌鉴权 + 静默刷新机制。
  7. 操作审计（v1.4.1）：全部关键写操作（创建订单/生产工单/采购单/委外单/入库单）均记录操作日志，日志查询接口限管理员访问。
  8. 双 Token 认证体系（v1.5.0）：access token 短效 2h + refresh token 长效 30d，采用独立密钥签发，降低令牌泄露风险。
  9. Helmet 安全响应头（v1.5.0）：防 XSS、点击劫持、MIME 噗探等常见 Web 攻击。
  10. Zod Schema 输入校验（v1.5.0）：8 个 Schema 覆盖全部创建接口，拒绝非法请求体。

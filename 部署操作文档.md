# 铭晟管理系统 - 部署与执行白皮书

> **系统版本:** v1.5.2  
> **更新要点:** v1.5.2 为代码质量优化版，完成了 ProcessConfigPanel 共享组件抽取（消除三处重复）、CSP 安全策略配置、产品编码唯一性校验、buildTree O(n) 优化、竞态修复、版本号自动注入等改进。前序版本已完成材质分类管理、材质匹配过滤、工序物料绑定与首道工序自动扣减库存、双 Token 认证、Helmet 安全头、Zod Schema 校验、全景看板、批次追溯等功能。

如果您需要将本旗舰版系统纯离线或在其他局域网服务器/开发机上全新部署，请严格遵循以下执行流：

---

## 1. 基础节点环境准备 (Node.js)
新电脑或服务器必须具备基础的 JavaScript 运行引擎：
- 访问 Node 官方下载站并安装 [Node.js](https://nodejs.org/zh-cn/)（推荐下载 v18.x 或 v20.x 的 LTS 长期支持版）。
- 安装过程中请一直点击“下一步”，**切勿更改**默认环境变量的注入配置。

![示意图: Node环境检查](https://via.placeholder.com/600x120.png?text=Verify+Installation:+node+-v)
✅ 验证步骤：按下 `Win + R` 键，输入 `cmd` 打开终端终端，敲入如下指令：
```bash
node -v
# 若输出版本号如 v18.17.0 等字样则环境绿灯放行。
```

## 2. 工程文件迁移与装载
- 将您本机的整个 `erp-mes-system` 工程文件夹通过 U 盘、坚果云、FTP 或网络传输等手段完整拷贝到目标电脑中。
- ⚡ **性能向提示**：为了极速地完成传输，可以在拷贝前**先删去**原有项目后端 `backend/node_modules` 与前端 `frontend/node_modules` 这两大依赖黑洞文件夹，体积可直降 90% 以上。

---

## 3. 全局总装与初始化（核心）
在目标机存好的项目根目录（例如 `D:\项目\erp-mes-system`）下：

![示意图: 一键执行安装卡片](https://via.placeholder.com/600x250.png?text=One-Click+Installation:+install.bat)

- 找到并双击运行 **`install.bat`** 引导器文件。
- 该装配脚本将如同自动驾驶般完成以下工作，并伴有进度反馈：
  1. 拦截并检测本机 Node 环境。
  2. 潜入 `backend` 目录，通过 `npm install` 为 Sqlite3 等本地持久层以及加解密引擎重新拉取编译对应的 Windows 二进制依赖。
  3. 潜入 `frontend` 目录拉取依赖，并无情触发 Vite 的树摇压缩编译流水线（`npm run build`）。
- 静待黑色命令窗提示绿色的 **“[√] 安装完成！”** 字样，即可按任意键全身而退。

---

## 4. 点火运行服务端
- 双击运行 **`start.bat`** 启动文件。
- 随即会唤醒一个常驻的黑色网关控制台进程，并在 3 秒内自动为您从默认浏览器弹开系统首页工作台。
- 🔗 默认路由为：`http://localhost:3198`

> ⚠️ **警告**：该控制台承载了所有 Node.js 业务接口、扫码监听、与轻量级 SQLite 数据落盘的线程，**请绝对不要关闭该黑窗口！如果将其叉掉，前台网页立刻宕机变“板砖”。**

![示意图: 成功运行截图](https://via.placeholder.com/600x300.png?text=Sever+Running...+%5B%E5%8B%BF%E5%85%B3%E9%97%AD%E6%AD%A4%E7%AA%97%E5%8F%A3%5D)

---

## 5. 优雅熄火与清理
- 当全天产能工作结束，或您想要关机、打扫环境时，请勿粗暴拔电。
- 双击项目目录下的 **`stop.bat`**。
- 它就像一把安全锁，能一键平滑寻址并终止相关 Node 监听端口，防止底层轻型数据库处于热更正被意外锁表，从而安稳地停止所有进程。

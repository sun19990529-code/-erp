## 第一部分：本地开发机部署

如果您需要将本旗舰版系统纯离线或在其他局域网服务器/开发机上全新部署，请严格遵循以下执行流：

### 1. 基础节点环境准备 (Node.js)
新电脑或服务器必须具备基础的 JavaScript 运行引擎：
- 访问 Node 官方下载站并安装 [Node.js](https://nodejs.org/zh-cn/)（推荐下载 v18.x 或 v20.x 的 LTS 长期支持版）。
- 安装过程中请一直点击"下一步"，**切勿更改**默认环境变量的注入配置。

✅ 验证步骤：按下 `Win + R` 键，输入 `cmd` 打开终端，敲入如下指令：
```bash
node -v
# 若输出版本号如 v18.17.0 等字样则环境绿灯放行。
```

### 2. 安装 PostgreSQL 数据库
- 下载 [PostgreSQL Windows 安装包](https://www.enterprisedb.com/downloads/postgres-postgresql-downloads)（推荐 16.x 版本）
- 安装过程中需记住以下设置：
  - **端口号**：`54321`（或使用默认 `5432`）
  - **超级用户密码**：设一个安全密码
- 安装完成后，打开 pgAdmin 或 psql 创建数据库：

```sql
CREATE DATABASE "msgy-erp" ENCODING 'UTF8';
```

✅ 验证步骤：
```bash
psql -U postgres -p 54321 -c "SELECT 1"
# 若输出 1 则 PostgreSQL 正常运行。
```

### 3. 工程文件迁移与装载
- 将整个 `erp-mes-system` 工程文件夹通过 U 盘、坚果云、FTP 或网络传输等手段完整拷贝到目标电脑中。
- ⚡ **性能向提示**：拷贝前**先删去** `backend/node_modules` 与 `frontend/node_modules`，体积可直降 90% 以上。

### 4. 配置数据库连接
在 `backend/` 目录下创建 `.env` 文件（可参考 `.env.example`）：

```ini
DB_HOST=localhost
DB_PORT=54321
DB_USER=postgres
DB_PASSWORD=你的PostgreSQL密码
DB_NAME=msgy-erp
```

> ⚠️ **生产环境**还需额外设置 JWT 密钥，详见第二部分。

### 5. 全局总装与初始化（核心）
- 找到并双击运行 **`install.bat`** 引导器文件。
- 该装配脚本将自动完成以下工作：
  1. 拦截并检测本机 Node 环境。
  2. 潜入 `backend` 目录拉取依赖。
  3. 潜入 `frontend` 目录拉取依赖并触发 Vite 编译（`npm run build`）。
- 静待提示 **"安装完成！"** 字样即可。

### 6. 初始化数据库表结构与基础数据
首次部署时需要导入数据：

```powershell
cd backend
# 方式一：从已有的 pg_dump 备份文件恢复
$env:PGPASSWORD = "你的密码"
pg_restore -h localhost -p 54321 -U postgres -d msgy-erp --clean --if-exists "备份文件路径.dump"

# 方式二：从旧 SQLite 数据库迁移（如果有 mes.db）
node scripts/migrate-data.js
```

### 7. 点火运行服务端
- **方式一（管理员启动，推荐）**：双击 **`server-start.ps1`**，自动请求 UAC 管理员提权，优先使用 PM2 托管服务
- **方式二（普通启动）**：双击运行 **`start.bat`**，直接启动 Node 进程
- 随即会唤醒常驻控制台进程，并自动打开浏览器。
- 🔗 默认路由为：`http://localhost:3198`

> ⚠️ **警告**：该控制台承载了所有业务接口，**请绝对不要关闭该黑窗口！**

### 8. 优雅熄火与清理
- 双击项目目录下的 **`stop.bat`**，平滑终止所有进程。

## 第二部分：远程服务器部署

适用于将系统部署到外网服务器（Windows Server / Windows 主机），实现从开发机一键推送更新。

### 1. 服务器首次准备

#### 1.1 安装 Node.js
同上述第一部分步骤 1。

#### 1.2 安装 PostgreSQL
同上述第一部分步骤 2。创建数据库 `msgy-erp`。

#### 1.3 初始化服务器环境
以 **管理员身份** 打开 PowerShell 运行：

```powershell
# 允许执行脚本（仅需一次）
Set-ExecutionPolicy RemoteSigned -Scope LocalMachine -Force

# 安装 PM2 进程管理器
npm install -g pm2
```

#### 1.4 配置 .env（关键！）

在服务器 `F:\erp-mes-system\backend\` 下创建 `.env`：

```ini
# PostgreSQL 连接
DB_HOST=localhost
DB_PORT=54321
DB_USER=postgres
DB_PASSWORD=服务器PG密码
DB_NAME=msgy-erp

# 生产环境（必须设置！服务会校验）
NODE_ENV=production
JWT_SECRET=一个足够复杂的随机字符串_至少32位
JWT_REFRESH_SECRET=另一个不同的随机字符串_至少32位
PORT=3198
```

> ⚠️ `JWT_SECRET` 和 `JWT_REFRESH_SECRET` 在生产环境下**必须设置且不能相同**，否则服务拒绝启动。

#### 1.5 首次部署与启动

```powershell
# 安装后端依赖
cd F:\erp-mes-system\backend
npm install --production

# 导入数据（从开发机导出的 dump 文件）
$env:PGPASSWORD = "服务器PG密码"
pg_restore -h localhost -p 54321 -U postgres -d msgy-erp --clean --if-exists "F:\erp-export.dump"

# 构建前端（首次需要）
cd ..\frontend
npm install
npm run build

# 用 PM2 启动服务
cd ..\backend
pm2 start server.js --name erp
pm2 save
```

#### 1.6 验证
- 浏览器访问 `http://localhost:3198`
- 默认账号：`admin` / 密码：`admin123`

### 2. 一键同步（日常更新）

开发机代码提交到 Git 后，在**服务器**上运行同步脚本：

```powershell
# 右键以管理员身份运行
.\server-sync.ps1
```

脚本自动完成 6 个步骤：

| 步骤 | 说明 |
|---|---|
| [1/6] Pull code | 从 Git 拉取最新代码 |
| [2/6] Check .env | 检查 PostgreSQL 配置是否完整 |
| [3/6] Frontend deps | 安装前端依赖 |
| [4/6] Build frontend | 编译前端 |
| [5/6] Backend deps | 安装后端依赖 |
| [6/6] Restart | PM2 自动重启服务 |

> **安全保障**：`.env` 文件和 PostgreSQL 数据库**不在 Git 管理范围内**，同步操作不会影响生产数据。

### 3. PM2 常用命令

在服务器 PowerShell 中使用：

| 命令 | 用途 |
|---|---|
| `pm2 status` | 查看服务运行状态 |
| `pm2 restart erp` | 重启服务 |
| `pm2 stop erp` | 停止服务 |
| `pm2 logs erp` | 查看运行日志 |
| `pm2 logs erp --lines 50` | 查看最近 50 行日志 |

### 4. 数据库备份与恢复

#### 备份
```powershell
# 方法一：通过系统内置备份功能（推荐）
# 浏览器 → 系统管理 → 数据库备份 → 点击"立即备份"

# 方法二：命令行手动备份
$env:PGPASSWORD = "你的PG密码"
pg_dump -h localhost -p 54321 -U postgres -F c -f "F:\备份\erp-$(Get-Date -Format 'yyyyMMdd').dump" msgy-erp
```

#### 恢复
```powershell
# 1. 先停服务
pm2 stop erp

# 2. 从备份还原
$env:PGPASSWORD = "你的PG密码"
pg_restore -h localhost -p 54321 -U postgres -d msgy-erp --clean --if-exists "F:\备份\erp-20260401.dump"

# 3. 重启服务
pm2 start erp
```

> 💡 与旧版 SQLite 不同，PostgreSQL 备份恢复**不需要**停服务也能执行，但建议停服以避免数据冲突。

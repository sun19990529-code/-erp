## 功能概览

系统支持两种机器人能力：

| 能力 | 说明 | 需要公网吗 |
|------|------|-----------|
| **📤 单向推送** | ERP 自动把告警/日报推送到群 | ❌ 不需要 (通过普通 Webhook 发送) |
| **🔄 双向交互 (钉钉)** | 群内 @机器人 发命令查询数据 | ✅ 需要公网 IP 供钉钉回叫 |
| **🤖 智能交互 (企微)** | 群内 @智能机器狗 查库存查单据 | ❌ 不需要 (通过官方长连接 SDK 直连) |

**已接入的告警推送：**
- 🔴 来料/巡检/委外检验不合格
- ⚠️ 库存低于安全水位
- ⚠️ 订单/采购/委外单超期
- 📊 每日 8:00 自动推送生产日报

## 钉钉机器人配置

### 2.1 推送机器人（单向通知）

> 用途：接收 ERP 系统主动推送的告警和日报

**步骤：**

```
1. 打开钉钉群 → 群设置（右上角 ⚙️）
2. 点击「智能群助手」→「添加机器人」
3. 选择「自定义（通过 Webhook 接入自定义服务）」
4. 设置机器人名称，如「铭晟ERP通知」
5. 安全设置选择「加签」，记录下 Secret 密钥
6. 完成后复制 Webhook 地址
```

> [!IMPORTANT]
> 复制两个关键信息：
> - **Webhook URL**: `https://oapi.dingtalk.com/robot/send?access_token=xxxx`
> - **加签 Secret**: `SECxxxx`

然后在后端 `.env` 文件中配置：

```env
DINGTALK_WEBHOOK_URL=https://oapi.dingtalk.com/robot/send?access_token=你的token
DINGTALK_SECRET=SEC你的加签密钥
```

重启后端服务即可生效。

---

### 2.2 交互机器人（双向查询）

> 用途：在群内 @机器人 输入命令，实时查询 ERP 数据

**前置条件：** ERP 后端服务器必须有公网可达的地址（如 `http://你的公网IP:3198`）

**步骤：**

```
1. 打开钉钉群 → 群设置 → 智能群助手 → 添加机器人
2. 选择「自定义（通过 Webhook 接入自定义服务）」
3. 勾选「是否开启 Outgoing 机制」（关键！）
4. 填写 POST 地址：

   http://你的公网IP:3198/api/bot/dingtalk

5. （可选）设置 Token 用于签名验证
6. 完成创建
```

> [!WARNING]
> **安全提醒：** 如果 ERP 服务直接暴露在公网，强烈建议：
> - 设置 Outgoing Token 进行签名验证
> - 使用 Nginx 反向代理，仅放行 `/api/bot/*` 路径
> - 启用 HTTPS

**Nginx 推荐配置（仅暴露机器人接口）：**

```nginx
server {
    listen 443 ssl;
    server_name erp.yourcompany.com;

    # 只转发机器人回调
    location /api/bot/ {
        proxy_pass http://127.0.0.1:3198;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 其他路径拒绝外部访问
    location / {
        return 403;
    }
}
```

## 企业微信机器人配置 (强推组合方案)

对于企业微信，我们采用**“普通群机器人管发送” + “长连接智能机器人管接单”**的双剑合璧模式，**且全程不需要公网暴露**，极其适合架设在本地工厂的内网服务器！

### 3.1 推送机器人（专管主动发日报、发告警）

```
1. 打开企业微信群 → 群设置
2. 点击「群机器人」→「添加」
3. 填写机器人名称，如「铭晟告警推送」
4. 创建后复制 Webhook 地址
```

在 `.env` 中配置：
```env
WECHAT_WEBHOOK_URL=https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=你的key
```

### 3.2 智能长连接机器人（专管群内 @ 查进度查库存）

> [!TIP]
> 企微传统的应用自建回调机制极其复杂。现在最新启用了由腾讯提供的**官方 WebSocket 智能机器人 SDK 长连接**。这意味着您的 ERP 就像登录微信电脑版一样，开机就悄悄连上企微，谁艾特它它都秒回！

```
1. 在企业微信工作台，打开并进入【智能机器人】管理端。
2. 创建机器人时，API 模式选择：「使用长连接」。
3. 保存后系统会生成并显示 Bot ID 和 Secret。
```

在 `.env` 中补充您的机器人凭证（由腾讯官方发放）：
```env
WECOM_BOT_ID=你的BotID
WECOM_BOT_SECRET=你的Secret
```

## 后端环境变量配置

编辑 `backend/.env` 文件，按需配置以下变量：

```env
# ==================== 推送播报机器人（选填）====================
# 发送每日报表、库存跌破告警（填入对应链接启用功能）

# 企业微信 Webhook
WECHAT_WEBHOOK_URL=https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=xxx

# 钉钉 Webhook
DINGTALK_WEBHOOK_URL=https://oapi.dingtalk.com/robot/send?access_token=xxx

# 钉钉加签密钥（推送机器人的安全验证）
DINGTALK_SECRET=SECxxxxx

# ==================== 答疑交互机器人（双向对话） ====================

# [重磅] 企微长连接智能狗 (支持无公网全内网环境自动响应!)
# 在此填入分配的机器人票据即可激活此服务：
WECOM_BOT_ID=aibnspqblIM_rXOrgIZi1A16rX5DFJcNzei
WECOM_BOT_SECRET=YDRS2VRnJhBM3D6K4AkVvGwgC1elIzVGPiQcJeKh8jr

# 钉钉 Outgoing 签名 Token（钉钉交互机器人的安全验证，可选）
DINGTALK_OUTGOING_TOKEN=你的outgoing签名token
```

> [!TIP]
> 可以同时配置钉钉和企业微信，告警会**同时推送到两个平台**。

配置完成后重启后端服务：
```powershell
cd d:\项目\erp-mes-system\backend
node server.js
```

启动时会看到：
```
生产日报定时推送已启动，下次推送: 2026/4/3 08:00:00
```

## 交互命令大全

在钉钉群中 @机器人 并发送以下命令：

### 📊 生产状态

```
@铭晟ERP 生产状态
```

返回当前生产概览，包括：
- 待排产订单数量
- 进行中工单数量
- 质检暂停工单
- 待检验数量
- 工单进度 TOP5（含进度条）
- 交期预警（3天内到期的订单）

---

### 🔍 查询工单

```
@铭晟ERP 工单 PO-20260401-001
```

返回指定工单的详细信息：
- 产品名称、规格
- 当前状态和进度百分比
- 各工序的完成情况
- 关联的销售订单

---

### 📦 查询订单

```
@铭晟ERP 订单 SO-20260401-001
```

返回指定订单的状态：
- 客户名称、总金额
- 交货日期
- 关联的生产工单及进度

---

### 📦 查询库存

```
@铭晟ERP 库存 六角螺栓
```

支持模糊搜索，返回匹配的产品库存信息：
- 产品编码、名称
- 当前库存数量
- 安全库存水位
- 预警状态（正常/不足）

---

### ⚠️ 超期检查

```
@铭晟ERP 超期检查
```

列出所有超期的订单和采购单。

---

### 📖 帮助

```
@铭晟ERP 帮助
```

显示所有可用命令列表。

## 定时日报配置

系统默认**每天早上 8:00** 自动推送生产日报到已配置的群。

日报内容与「生产状态」命令相同，包含：

```
📊 铭晟ERP 生产状态 (2026-04-02)

| 指标          | 数量 |
|---------------|------|
| 📦 待排产订单 | 3    |
| 🔧 进行中工单 | 5    |
| ⚠️ 质检暂停   | 1    |
| 📋 待检验     | 2    |

🔝 工单进度:
1. PO-20260401-001 六角螺栓 M12  ████████░░ 80%
2. PO-20260331-003 无缝钢管      ██████░░░░ 60%

⚠️ 交期预警:
- SO-20260328-001 示例客户 🔴已超期
- SO-20260401-002 另一客户 🟡2天后到期
```

> [!NOTE]
> 如需修改推送时间（如改为 9:00），需要修改 `server.js` 中 `scheduleDailyReport` 函数里的 `next8am.setHours(8, 0, 0, 0)` 改为 `next8am.setHours(9, 0, 0, 0)`。

## 常见问题排查

### Q1: 群里收不到推送通知？

**排查步骤：**
1. 检查 `.env` 中 `DINGTALK_WEBHOOK_URL` 或 `WECHAT_WEBHOOK_URL` 是否正确
2. 查看后端控制台是否有 `[webhook/dingtalk]` 或 `[webhook/wechat]` 相关日志
3. 确认钉钉机器人安全设置中的「加签」密钥已正确配置到 `DINGTALK_SECRET`

### Q2: @机器人 没有回复？

**排查步骤：**
1. 确认创建的是 **Outgoing 机器人**（非普通 Webhook 机器人）
2. 确认回调地址 `http://公网IP:3198/api/bot/dingtalk` 可从外网访问
3. 在浏览器测试：`http://公网IP:3198/api/bot/query?q=帮助`，确认接口正常返回
4. 查看后端控制台是否有 `[bot/dingtalk]` 相关日志
5. 检查防火墙是否放行了 3198 端口

### Q3: 机器人回复「签名验证失败」？

检查 `.env` 中 `DINGTALK_OUTGOING_TOKEN` 是否与钉钉机器人设置中的 Token 一致。如果不需要签名验证，可以不设置此变量。

### Q4: 日报没有按时推送？

1. 确认后端服务启动时有 `生产日报定时推送已启动` 日志
2. 日报使用 `setTimeout` + `setInterval` 实现，如果重启了服务器，计时器会重新计算
3. 如果服务在 8:00 前后重启，可能会跳过当天的推送

### Q5: 如何手动触发日报推送？

可以在浏览器或 curl 中访问：
```
GET http://localhost:3198/api/bot/query?q=日报
```

或者在群里 @机器人 发送 `日报`。

### Q6: 频率限制？

钉钉和企业微信的群机器人都有**每分钟 20 条**的频率限制。系统已对超期检查做了消息合并处理，正常使用不会触发限制。
